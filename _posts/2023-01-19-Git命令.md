### Git描述   
 
 Git是一个免费的、开源的分布式*版本控制*系统，可以快速高效地处理从小型到大型的项目。
 
### Git常用命令
 > git 在文件夹中进行初始化git init
 
 > git 本地分为三个区：工作区->暂存区->本地库
 
 > 平时coding都在工作区，add命令将工作区的代码提交到暂存区，在idea中一般自动添加git add . 
 >> 注意‘.’，代表所有更改/添加的文件，commit 命令 将更改保存到本地库, -m 输入改动说明git commit -m "msg"

 > reset命令：前进或者后退历史版本# hard：本地库的指针移动的同时，重置暂存区，重置工作区# mixed：本地库的指针移动的同时，重置暂存区，但是工作区不动# soft: 本地库的指针移动的时候，暂存区，工作区都不动git reset --hard/mixed/soft “relog的索引”
 
 > diff命令 将工作区中的文件和暂存区中文件进行比较git diff [文件名]

### Git分支操作
 > 将仓库代码拉到本地库

 `git clone -b branch_name ssh_location`
 > 查看本地分支

 `git branch`
 > 带 * 分支为当前分支

 > 创建新的分支

 `git branch new_branch_name`
 > 转换到新的分支上

 `git checkout new_branch_name`
 > 创建分支+切换到新分支

 `git checkout -b new_branch_name`
 > 将分支信息推送到远程仓库

 `git push origin(别名) HEAD -u`
 
 `git push origin new_branch_name`
 > 获取当前分支最新代码

 `git pull`

 > 分支合并(已尝试)

 `git checkout master`
 
 `git merge branch_name`

### Git多账户配置
##### 1、生成多个SSH密钥
```bash
ssh-keygen -t rsa -C "youremail@example.com" -f ~/.ssh/github_rsa_account
ssh-keygen -t rsa -C "youremail@example.com" -f ~/.ssh/gitlab_rsa_account
```
结果生成的 `github_rsa_account.pub`、`github_rsa_account`、`gitlab_rsa_account.pub`、`gitlab_rsa_account`， 一个命令生成2个对应的文件；
> ps：`rsa`只是一个加密手段，例如还有`ed25519`等
##### 2、配置 SSH 客户端
路径：`~/.ssh/config`
修改`config`配置内容
```
Host gitlab
    HostName code.amh-group.com
    User jiakai.wu@amh-group.com
    IdentityFile ~/.ssh/gitlab_ed25519 
    AddKeysToAgent yes
    UseKeychain yes

Host github
    HostName github.com
    User 852094958@qq.com
    IdentityFile ~/.ssh/github_rsa
    AddKeysToAgent yes
    UseKeychain yes
```
#### 3、添加公钥到 Git 账户
将生成的公钥（`.pub` 文件）内容复制到对应的 Git 账户中
```bash
cat ~/.ssh/github_rsa_account.pub
cat ~/.ssh/gitlab_rsa_account.pub
```
然后在 GitHub、GitLab 或 Gitee 等平台的 SSH 密钥管理页面中添加这些公钥。
#### 4、新增 ssh-key到ssh-agent
```bash
sh-add -K ~/.ssh/github_rsa_account
sh-add -K ~/.ssh/gitlab_rsa_account
```
#### 5、连接测试
```bash
ssh -T git@github.com-account1
ssh -T git@gitlab.com-account2
```
如果连接成功，会显示类似以下的提示：
`Hi username! You've successfully authenticated, but GitHub does not provide shell access.`
#### 6、配置 Git 仓库
在每个项目的本地仓库中，设置局部的用户名和邮箱
```bash
cd /path/to/project1
git config user.name "用户名1"
git config user.email "邮箱1"

cd /path/to/project2
git config user.name "用户名2"
git config user.email "邮箱2"
```

#### 注意事项
- 确保每个项目的 `.git/config` 文件中使用了正确的 SSH 别名（如 `github.com-account1`）。
- 如果使用 HTTPS 方式克隆仓库，需要在 Git 配置中指定正确的用户名和邮箱。
通过以上步骤，你可以在 macOS 上配置多个 Git 账户，并在不同的项目中使用不同的账户进行操作。
